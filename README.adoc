= Local AI Setup Guide: Llama.cpp & Claw
:toc: left
:source-highlighter: highlight.js
:icons: font

This guide covers the installation, model acquisition, and execution of local LLMs using `llama.cpp` and the `claw-code` tool on Ubuntu, optimized for hardware like the Ryzen AI 9 HX-370.

== 1. Prerequisites & Installation

First, ensure the system is up-to-date and install the necessary build tools and dependencies.

[source,bash]
----
# Update system and install build tools
sudo apt update && sudo apt install -y \
    build-essential \
    cmake \
    git \
    git-lfs \
    python3-pip

# If your PC supports Vulkan, install the necessary Vulkan tools and drivers for GPU acceleration
sudo apt install vulkan-tools mesa-vulkan-drivers

# Install Hugging Face CLI for model management
pip install huggingface_hub --break-system-packages
----

=== Hugging Face Authentication
To download gated models or use private settings, authenticate with your token:
[source,bash]
----
# Get token from: https://huggingface.co/settings/tokens
huggingface-cli login
----

== 2. Model Downloads

We use GGUF formatted models for compatibility with `llama.cpp`.

You can click "Copy modle name to clipboard" and download it with the scripts/download_models.sh script, or use the Hugging Face CLI directly:

[source,bash]
----
./scripts/download_models.sh unsloth/Phi-4-mini-instruct-GGUF
----

Or use the Hugging Face CLI directly to specify models and output directories:

[source,bash]
----
# Qwen 3.5 9B (General Purpose)
huggingface-cli download unsloth/Qwen3.5-9B-GGUF \
    Qwen3.5-9B-Q6_K.gguf --local-dir ./models/qwen3.5-9b
----

== 3. Running Llama.cpp

=== CLI Test Mode
Run a quick test to ensure Vulkan acceleration is working on your Ryzen AI chip.

Run the **./scripts/generate_start_scripts.sh** script to create start scripts for your downloaded models, then execute one of the generated scripts. The output script will look something like this, if you not have vulkan you can remove the --vulkan flag and run it on CPU, but performance will be much slower.

=== Server Mode
Start the OpenAI-compatible API server by running the script of your chosen model, that was generated by the **./scripts/generate_start_scripts.sh** script.

Make sure that you not run multiple servers at the same time, or have enopugh RAM avaiable.

=== Verify API Connectivity
[source,bash]
----
curl -X POST http://127.0.0.1:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-local-coding" \
  -d '{
    "model": "qwen3-local",
    "messages": [{"role": "user", "content": "say hello"}],
    "max_tokens": 50
  }'
----

.Table of llama-server Sampling Parameters
[cols="1,1,1,2"]
|===
|Parameter |Flag |Example |Effect

|Temperature
|`--temp`
|0.7
|Randomness control (0.3-0.8 for speed/focus)

|Top-k
|`--top-k`
|10
|Max tokens considered from probability distribution

|Top-p
|`--top-p`
|0.9
|Cumulative probability threshold (nucleus sampling)

|Min-p
|`--min-p`
|0.05
|Relative probability floor for token consideration

|Sampler Order
|`--samplers`
|`"top_k;top_p;min_p;temperature"`
|Sequence of sampler application

|Max Tokens
|`--n-predict`
|512
|Output token limit (-1 = unlimited)

|Advanced Samplers
|`--typ_p`, `--xtc_*`
|Varies
|Typ_p=0.25, XTC probability/threshold controls
|===

=== Web Interface
Open http://127.0.0.1:8082/ in your browser to access the web interface for testing and interacting with your local LLM.
Enter the key `sk-local-coding`
You will be presented with a sufisticated web interface to interact with your local LLM, test prompts, and view responses in real-time,
ChatGpt style with more advanced features like system settings.

== 4. Claw Code Tooling

`Claw` acts as a bridge for coding tasks using your local API.

=== Build
[source,bash]
----
cd claw-code/
./install.sh --release
----

=== Configuration & Execution
Ensure the binary exists at `rust/target/release/claw`, then export your environment variables.

[source,bash]
----
export OPENAI_BASE_URL="http://127.0.0.1:8080/v1"
export OPENAI_API_KEY="sk-local-coding"

Check status
./target/release/claw --model openai/qwen3-local doctor

Run a prompt
./target/release/claw --model openai/qwen3-local prompt "Write a rust function to sort a list."
----

Add the `export` lines to your `.bashrc` or `.zshrc` to make them persistent across sessions.
