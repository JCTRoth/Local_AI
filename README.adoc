= Local AI Setup Guide: Ryzen AI NPU & Claw
:toc: left
:source-highlighter: highlight.js
:icons: font

This guide covers the installation, model acquisition, and execution of local LLMs using the Ryzen AI NPU launcher and the `claw-code` tool on Ubuntu, optimized for hardware like the Ryzen AI 9 HX-370 on my Ubuntu based system.

When using the scripts in the script folder and the from there generated starter scripts, you are able to start every model your PC can handle and can try out different models without changing the configuration of the tools.

No matter wich script you now start form the script folder, continue will directly start to use it.




== 1. Prerequisites & Installation

First, ensure the system is up-to-date and install the necessary build tools and dependencies.

[source,bash]
----
# Update system and install build tools
sudo apt update && sudo apt install -y \
    build-essential \
    cmake \
    git \
    git-lfs \
    python3-pip

# If your PC supports Vulkan, install the necessary Vulkan tools and drivers for GPU acceleration
sudo apt install vulkan-tools mesa-vulkan-drivers

# Install Hugging Face CLI for model management
pip install huggingface_hub --break-system-packages
----

=== CLI Test Mode
Run a quick test to ensure the Ryzen AI NPU stack is working on your chip.

Use the bundled autocomplete starter script or call the launcher directly for the included ONNX model:

[source,bash]
----
./npu/run_model.sh start --port 8081 ./npu/models/Qwen2.5-Coder-0.5B-Instruct_rai_1.7.1_npu_4K/
----

=== Server Mode
Start the OpenAI-compatible API server by launching the Ryzen AI model folder with `./npu/run_model.sh`.

The launcher writes logs inside the model directory and records the chosen port and PID there.

Make sure that you do not run multiple servers at the same time, or have enough RAM available.

=== Verify API Connectivity

== 4. Claw Code Tooling

`Claw` acts as a bridge for coding tasks using your local API.

=== Build
[source,bash]
----
cd claw-code/
./install.sh --release
----


=== Configuration & Execution
Ensure the binary exists at `rust/target/release/claw`, then export your environment variables.

[source,bash]
----
export OPENAI_BASE_URL="http://127.0.0.1:8080/v1"
export OPENAI_API_KEY="sk-local-coding"

Check status
./target/release/claw --model openai/qwen3-local doctor

Run a prompt
./target/release/claw --model openai/qwen3-local prompt "Write a rust function to sort a list."
----

Add the `export` lines to your `.bashrc` or `.zshrc` to make them persistent across sessions.

== Using the NPU

The `starter_scripts/autocomplete_model` wrapper starts the bundled Ryzen AI autocomplete model.

For Ryzen AI / ONNX GenAI model folders use `./npu/run_model.sh` directly.

`./npu/run_model.sh` now:

- starts a model from a model path
- writes logs to `logs/<unix_timestamp>_<model_name>.log` inside the model folder
- records the chosen port and pid in the model folder
- can list running managed model processes
- can report current NPU usage and managed model processes with `stats`
- can stop a model by pid, port or model path
- refuses startup if NPU usage is above `70%`

Important: AMD's Linux documentation currently describes an LLM benchmark/deployment flow, not a built-in port-based API server. If `ryzenai-server` is not installed, you must provide a backend command explicitly with `--backend-command` or `LOCAL_AI_NPU_BACKEND_COMMAND`.

----
# Start a Ryzen AI model folder
./npu/run_model.sh ./npu/models/Qwen2.5-Coder-0.5B-Instruct_rai_1.7.1_npu_4K/

# Show managed model processes
./npu/run_model.sh list

# Show NPU usage and managed model processes
./npu/run_model.sh stats

# Stop a model by port
./npu/run_model.sh stop --port 8080
----
